# Security Policy

## Supported Versions

Only the current `main` branch receives security fixes.

| Version | Supported |
|---------|-----------|
| main    | Yes       |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.** Public disclosure before a fix is available puts other users at risk.

Instead, report vulnerabilities by email. Include:

- A description of the vulnerability
- Steps to reproduce it
- The potential impact (e.g., unauthorized access, data exposure)
- Any suggested fix, if you have one

You can reach the maintainer at the email address listed on the GitHub profile.

**Response timeline:**

- Acknowledgement within 72 hours
- Status update within 7 days
- Patch target within 14 days for critical issues

Once a fix is released, the vulnerability will be disclosed in the release notes.
